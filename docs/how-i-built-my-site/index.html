How I built my site

There are many different ways to build a website these days. Each and every way has its own advantages and disadvantages. Also, no developer can know every single way build the perfect website. On this page I will try to explain why I built this site the way I did. Spoiler alert: I used Drupal.

As you can read on the about me page, I didn't really like programming all that much when I left school. It was only after getting a job that I discovered programming could be fun when you actually kinda knew what you were doing. On my first job at volta.be we did most of our programming in Drupal. This is where I really started to appreciate best practices and object oriented programming. 

Learning and experimenting

I still remember being overwhelmed with new terminology and concepts when first starting out with Drupal. It took me a few weeks to figure out what I the hell I was actually doing. Actually, I still don't know exactly what I am doing. And this is why I decided to build this site in Drupal. Drupal 9 to be precise. While I feel very confident in using Drupal to build all kinds of crazy websites, there are still so much more features that I haven't got to use. I feel like I only just scratched the surface of what could be done with a powerful CMS like Drupal. My personal site gives me the perfect opportunity the experiment with new contrib modules, custom code and fancy ideas without breaking any off my clients live production websites. And that's what this site is all about. Learning new stuff while experimenting, breaking things and fixing things back again. 

How did I do it

But how did I actually built this site? What modules did I use, what custom code did I write, where am I hosting my code? We'll split this up in different sections.

Custom code

Automatic responsive images with lazy-loading

Drupal 9 has a very powerful media managing system out of the box. However, I find that you have to configure too much to get things working. You can read more about this in my blog. Long story short, I decided to build a custom module where you only have to configure the ratio in a fieldformmater and that's it. You can read the detailed working of this module in the same blog.
