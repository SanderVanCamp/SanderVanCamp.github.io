How to enable Xdebug on Lando

Fri, 11/06/2020 - 12:07

When developing a website or any other application being able to debug your code is a must. Whether it is just checking a value inside a variable or trying to figure out why your performance is so bad, every developer should know how to debug. So, how does this work when using Lando and PhpStorm? 


Lando is a free, op source Docker driven local development tool that is easy and fast to set up. But getting XDebug to work can be a real pain in the *ss. However, once figured out, you can set it up in a few simple steps. 

1. Prerequisites

  1. Make sure you have installed the Chrome Xdebug Helper
  2. Make sure you have configured PhpStorm to watch for xdebug on port 9000

2. In your .lando.yml file

Make sure you enable xdebug in your appserver and set a custom config file for php. Tip: make sure the path is correct. In this example the .lando folder is situated in the same directory as the .lando.yml file.

name: sandervancamp.be
recipe: drupal8
config:
  webroot: web
services:
  appserver:
    xdebug: true
    config:
      php: .lando/php.ini

3. In your .lando/php.ini file

So what's important in this file? Well, everything basically. 

xdebug.max_nesting_level = 256
xdebug.show_exception_trace = 0
xdebug.collect_params = 0
xdebug.mode = debug
xdebug.client_host = ${LANDO_HOST_IP}
xdebug.client_port = 9000
xdebug.start_with_request = yes
xdebug.log = /tmp/xdebug.log

4. Enjoy debugging

That's it! Everything should work just fine. Be sure to turn on your xdebug helper and your little green telephone in PhpStorm and you should be good to go!
